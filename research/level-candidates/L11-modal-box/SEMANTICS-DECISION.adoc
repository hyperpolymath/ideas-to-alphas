== The question, as posed

`MOTIVATION.adoc` framed it as a binary: *S4 global necessity* versus *contextual
modal types* (Nanevski–Pfenning–Pientka). Both were assumed to be candidate
readings of the same `Box`.

== The question was malformed

The two options are not rivals. They refine `□` along *different axes*:

* *CMTT* refines *which variables* a box may depend on (`[Ψ]A` — context).
* *Graded necessity* refines *how many times* its content may be used (`□_r A` — quantity).

A system can have both (`[Ψ]_r A`). So "S4 or contextual?" has no answer as
asked. The answerable question is: *which axis does L11 need?* L11 exists to
discipline the *usage* of persistent resources. That is the quantity axis.

== Decision

*Adopt graded necessity `□_r A`*, `r` drawn from the L10 multiplicity semiring
`{0, 1, ω}` (parameterised over any ordered semiring — see `QTT-INTEGRATION.adoc` §2).

Rejected: strict S4; contextual modal types.

== Rationale

=== 1. Strict S4 is refuted by L11's own worked example

S4 promotion requires a closed (or all-modal) context. `Modal.idr`'s motivating
function captures an ordinary variable:

[source,idris]
----
openPool : String -> Box Conn
openPool uri = mkBox (MkConn uri)     -- `uri` is an ordinary hypothesis
----

Under an honest strict-S4 reading this is *ill-typed*. The sketch never noticed
because `Box` was never given a promotion rule at all — `mkBox : a -> Box a` is
unrestricted, which is not S4's `□` and not anything else with a modal metatheory.
S4 was never actually on the table; it was a label on an unruled constructor.

=== 2. CMTT fixes the capture problem along an axis L11 does not need

CMTT *would* make `openPool` typecheck, as `[uri:String]Conn` — a *recipe* for a
connection awaiting a URI. But `[Ψ]A` carries no multiplicity anywhere. It would
buy well-typedness and contribute exactly nothing to the discipline L11 was created
to enforce. CMTT describes *code*; L11 describes a *resource*.

=== 3. Graded necessity fixes it on the right axis

Promotion by context scaling — `Γ ⊢ a : A` gives `r · Γ ⊢ box_r a : □_r A` —
permits capture and *charges* for it. `openPool` is well-typed at `r = ω` because
`ω · (uri :^1 String) = uri :^ω String`, and `String` is duplicable. The rule that
makes the example legal is the same rule that accounts for its cost. That is the
property L11 wanted.

=== 4. It composes with what `typell` already has

L10 already implements the `{0,1,ω}` semiring. Graded necessity is defined *from*
that semiring — `split` is `+`, `comult` is `·`, `drop` is `0`, `extract` is `1`.
L11 therefore adds a modality over existing algebra rather than importing a second,
unrelated metatheory. CMTT would import contexts-as-types, metavariables, and
contextual substitution — a large new surface for no usage-discipline gain.

=== 5. It is the estate's existing idiom

`epistemic-types/docs/secret-types.adoc` already classifies `S ℓ A` as a "graded
comonad / coeffect graded by clearance", and `EchoBridge.agda` grades `Echo r A`
over a tropical semiring with the algebra proved in `trope-checker` (Lean 4).
Grading modalities by a semiring is how this estate already reasons. `□_r` is the
usage-axis member of a family that exists, not a new invention.

=== 6. S4 is not lost

`□_ω` recovers the S4/`!`-style unrestricted box (`ω + ω = ω`, `ω · ω = ω`). The
decision *generalises* S4 rather than discarding it — everything the sketch can
currently express remains expressible, as the `r = ω` instance.

== Consequences [[consequences]]

=== `Modal.idr` must be rewritten (new blocker)

`QTT-INTEGRATION.adoc` §5 proves that the current

[source,idris]
----
dup : Box a -> (Box a, Box a)
----

is admissible only when `r + r = r`, hence only at `r ∈ {0, ω}`; and `r = 0` is
erased, so *every useful instance pins `r = ω`*. Consequences:

* `Box a` as written is isomorphic to *an L10 value at multiplicity ω*. L11
  currently adds **nothing** over L10.
* The proved `comonadLaw1`/`comonadLaw2` are sound but are the *ω-instance* laws.
  They are not evidence for the discipline at `r = 1`.
* `dup` must be replaced by `split : Box (r + s) a -> (Box r a, Box s a)`.

This is a *new* graduation blocker that did not exist before this decision — not
because anything regressed, but because the defect was previously invisible. Better
found here than in `typell`.

Honest summary: *2 of 6 done (one of them only as of today), 1 spec'd, 3 not
started — and the spec added a seventh item (the rewrite).* Graduation is **not**
close. This decision moved it further away in calendar terms while moving it
closer in truth: one checklist item that was recorded as complete turned out never
to have compiled, and the level's central construct turned out to be L10 in
disguise. The `route-to-mvp` milestone "First graduated artefact promoted to
typell" stays at 0% and should not move until `split` replaces `dup` and the laws
are re-derived at general `r`.

[IMPORTANT]
.The meta-finding
====
The `dup`/`Prelude.dup` clash meant `Modal.idr` never compiled, yet it was recorded
as proved at 100% for three months. The cause was not carelessness in the proof —
it was that *nothing could check it*: Idris2 was never installed (`IDRIS2_PREFIX`
and `PACK_DIR` pointed at a `pack` tree that did not exist), and CI gates only
`a-sounder-constitution`, not `research/`. An unrunnable proof gate produces
confident false records. Fix the gate before trusting any other research milestone
in this repo.
====

=== CMTT is shelved, not discarded

Contextual modal types remain the right tool for a *future* level whose boxed
object is genuinely open — staged compilation, a WasmGC code template with holes, a
typed-metavariable/tactic layer. That is a plausible later candidate. It is not L11.

== Alternatives considered

[cols="1,3",options="header"]
|===
| Alternative | Why rejected

| Strict S4 `□A`
| Refuted by `openPool` (capture). Would force pools to be closed, defeating the purpose.

| Contextual `[Ψ]A`
| Right fix, wrong axis. No multiplicity content; large metatheory import for zero usage-discipline gain.

| Bigraded `[Ψ]_r A`
| Both axes at once. No demonstrated L11 need for the context axis; pay for it when a level needs open boxes.

| Keep `Box` ungraded, document `r = ω`
| Honest, but then L11 is not a level — it is L10 usage with a wrapper. Rejected as it forfeits the level's reason to exist.

| Grade over the tropical semiring (reuse `EchoBridge`)
| Category error. Tropical grades measure *recoverability*, not *usage*. See `QTT-INTEGRATION.adoc` §1.
|===
